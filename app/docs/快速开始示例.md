# 快速开始示例

本示例带你快速体验规则配置化的完整流程：调整规则 → 验证结果 → 对比差异。

## 场景：调整旺衰判定规则

假设我们发现某个案例的旺衰判定结果不符合命理专业判断，需要调整规则。

### 步骤 1：查看当前结果

```bash
# 验证案例"蔡蔡"的旺衰判定
node verify-rule.mjs --case 蔡蔡 --module wangShuai > output/before.json

# 查看当前结果
cat output/before.json
```

输出示例：
```json
{
  "case": "蔡蔡",
  "module": "wangShuai",
  "result": {
    "dayMaster": "乙木",
    "season": "寅月（春季）",
    "deLing": true,
    "deDi": true,
    "deShi": true,
    "conclusion": "极旺"
  }
}
```

### 步骤 2：调整规则

编辑 `src/engine/rules/wangShuaiRules.json`：

```json
{
  "version": "1.0.0",
  "rules": {
    "verdictTable": {
      "得令+得地+得势": "极旺",
      "得令+得地+不得势": "偏旺",  // 修改这里：原来是"极旺"，改为"偏旺"
      "得令+不得地+得势": "偏旺",
      // ...
    }
  }
}
```

### 步骤 3：验证调整结果

```bash
# 再次验证
node verify-rule.mjs --case 蔡蔡 --module wangShuai > output/after.json

# 查看新结果
cat output/after.json
```

输出示例：
```json
{
  "case": "蔡蔡",
  "module": "wangShuai",
  "result": {
    "dayMaster": "乙木",
    "season": "寅月（春季）",
    "deLing": true,
    "deDi": true,
    "deShi": false,  // 注意：这里变成了 false
    "conclusion": "偏旺"  // 结论从"极旺"变为"偏旺"
  }
}
```

### 步骤 4：对比差异

```bash
# 对比修改前后的输出
node verify-rule.mjs --diff --before output/before.json --after output/after.json
```

输出示例：
```
差异对比:

旺衰判定:
- 修改前: 极旺
- 修改后: 偏旺
- 差异: 结论从"极旺"变为"偏旺"

得势判定:
- 修改前: true
- 修改后: false
- 差异: 得势判定从"true"变为"false"
```

### 步骤 5：确认调整是否符合预期

如果调整结果符合预期，则规则调整完成。如果不符合预期，继续调整规则并重复步骤 2-4。

## 场景：批量验证多个案例

假设我们需要验证规则调整对多个案例的影响。

### 步骤 1：批量验证修改前的结果

```bash
# 创建输出目录
mkdir -p output/before output/after

# 批量验证多个案例
for case in 蔡蔡 张三 李四; do
  echo "验证案例: $case"
  node verify-rule.mjs --case $case > output/before/$case.json
done
```

### 步骤 2：调整规则

编辑对应的规则文件，例如 `src/engine/rules/wangShuaiRules.json`。

### 步骤 3：批量验证修改后的结果

```bash
# 批量验证多个案例
for case in 蔡蔡 张三 李四; do
  echo "验证案例: $case"
  node verify-rule.mjs --case $case > output/after/$case.json
done
```

### 步骤 4：对比所有案例的差异

```bash
# 对比所有案例的输出
for case in 蔡蔡 张三 李四; do
  echo "=== 对比案例: $case ==="
  node verify-rule.mjs --diff --before output/before/$case.json --after output/after/$case.json
  echo ""
done
```

输出示例：
```
=== 对比案例: 蔡蔡 ===

差异对比:

旺衰判定:
- 修改前: 极旺
- 修改后: 偏旺
- 差异: 结论从"极旺"变为"偏旺"

=== 对比案例: 张三 ===

差异对比:

旺衰判定:
- 修改前: 偏旺
- 修改后: 偏旺
- 差异: 无变化

=== 对比案例: 李四 ===

差异对比:

旺衰判定:
- 修改前: 极弱
- 修改后: 极弱
- 差异: 无变化
```

## 场景：检测规则冲突

假设我们想检测规则配置中是否存在冲突。

```bash
# 检测规则冲突
node verify-rule.mjs --conflicts
```

输出示例：
```
规则冲突检测:

发现冲突:
1. 旺衰判定规则冲突
   - 规则A: 得令+得地+不得势 → 偏旺
   - 规则B: 得令+不得地+得势 → 偏旺
   - 冲突: 相同的条件组合有不同的结论

2. 用神选取规则冲突
   - 规则A: 身旺财弱 → 用财，忌比劫
   - 规则B: 身旺财弱 → 用火，忌金
   - 冲突: 相同的条件有不同的用神建议

建议:
1. 检查旺衰判定规则的条件组合，确保每个条件组合只有一个结论
2. 检查用神选取规则的优先级，确保规则顺序正确
```

根据冲突检测结果，我们可以：
- 修复规则冲突，确保规则配置正确
- 或者调整规则优先级，避免冲突
