# 八字项目规则配置化使用指南

## 概述

本指南帮助你快速上手八字项目的规则配置化方案，实现"快速调整、快速校验、快速感知调整结果"的目标。

## 核心架构

```
app/
├── src/
│   └── engine/
│       ├── rules/              # 规则配置目录
│       │   ├── wangShuaiRules.json      # 旺衰判定规则
│       │   ├── yongShenRules.json       # 用神选取规则
│       │   ├── geJuRules.json           # 格局判定规则
│       │   ├── shenShaRules.json        # 神煞判定规则
│       │   ├── shifuReplyTemplates.json # 先生回话模板
│       │   └── consumerReportRules.json # 消费者报告转化规则
│       ├── rulesLoader.ts      # 规则加载器（单例，支持缓存和热重载）
│       ├── wangShuaiAnalyzer.ts
│       ├── yongShenAnalyzer.ts
│       ├── geJuAnalyzer.ts
│       ├── shenShaAnalyzer.ts
│       ├── shifuEngine.ts
│       └── consumerReportGenerator.ts
└── verify-rule.mjs            # 命令行验证工具
```

## 快速开始

### 1. 调整规则

直接编辑对应的 JSON 配置文件：

```bash
# 编辑旺衰判定规则
vim src/engine/rules/wangShuaiRules.json

# 编辑用神选取规则
vim src/engine/rules/yongShenRules.json

# 编辑格局判定规则
vim src/engine/rules/geJuRules.json

# 编辑神煞判定规则
vim src/engine/rules/shenShaRules.json

# 编辑先生回话模板
vim src/engine/rules/shifuReplyTemplates.json

# 编辑消费者报告转化规则
vim src/engine/rules/consumerReportRules.json
```

### 2. 验证调整结果

使用命令行工具快速验证：

```bash
# 验证指定案例的旺衰判定
node verify-rule.mjs --case 蔡蔡 --module wangShuai

# 验证指定案例的用神选取
node verify-rule.mjs --case 蔡蔡 --module yongShen

# 验证指定案例的格局判定
node verify-rule.mjs --case 蔡蔡 --module geJu

# 验证指定案例的神煞判定
node verify-rule.mjs --case 蔡蔡 --module shenSha

# 验证完整分析结果
node verify-rule.mjs --case 蔡蔡
```

### 3. 对比输出差异

```bash
# 对比修改前后的输出
node verify-rule.mjs --diff --before output/before.json --after output/after.json
```

### 4. 查看覆盖率报告

```bash
# 生成规则覆盖率报告
node verify-rule.mjs --coverage
```

### 5. 检测规则冲突

```bash
# 检测规则冲突
node verify-rule.mjs --conflicts
```

## 规则配置文件说明

### wangShuaiRules.json（旺衰判定规则）

```json
{
  "version": "1.0.0",
  "rules": {
    "wangXiangTable": {
      "木": {
        "春": "旺",
        "夏": "休",
        "秋": "囚",
        "冬": "相",
        "季月": "死"
      },
      // ...
    },
    "deLingThreshold": {
      "得令": 3,
      "不得令": 2
    },
    "verdictTable": {
      "得令+得地+得势": "极旺",
      "得令+得地+不得势": "偏旺",
      // ...
    }
  }
}
```

**可调整项：**
- `wangXiangTable`: 五行在不同季节的旺相休囚死状态
- `deLingThreshold`: 得令判定的阈值
- `verdictTable`: 综合判断的投票表

## 验证工具详细说明

### verify-rule.mjs 命令行参数

```bash
node verify-rule.mjs [选项]

选项:
  --case <案例名>         指定要验证的案例（如：蔡蔡、张三）
  --module <模块名>       指定要验证的模块（wangShuai/yongShen/geJu/shenSha）
  --diff                 对比模式，需要 --before 和 --after 参数
  --before <文件路径>    修改前的输出文件
  --after <文件路径>     修改后的输出文件
  --coverage             生成规则覆盖率报告
  --conflicts            检测规则冲突
  --help                 显示帮助信息
```

## 常见使用场景

### 场景 1：调整旺衰判定阈值

**问题：** 某些案例的旺衰判定结果不符合命理专业判断。

**解决方案：**
1. 编辑 `wangShuaiRules.json`，调整 `deLingThreshold` 或 `verdictTable`
2. 运行验证工具：`node verify-rule.mjs --case 案例名 --module wangShuai`
3. 查看输出结果，确认是否符合预期
4. 如果不符合，继续调整规则，重复步骤 2-3

### 场景 2：优化先生回话文案

**问题：** 先生回话的文案不够准确或不够有温度。

**解决方案：**
1. 编辑 `shifuReplyTemplates.json`，修改对应的模板
2. 运行验证工具：`node verify-rule.mjs --case 案例名`
3. 查看先生回话输出，确认文案是否符合预期
4. 如果不符合，继续调整模板，重复步骤 2-3

### 场景 3：修改消费者报告的行业建议

**问题：** 消费者报告中推荐行业不够准确。

**解决方案：**
1. 编辑 `consumerReportRules.json`，修改 `wuxingGuide` 中的 `industries`
2. 运行验证工具：`node verify-rule.mjs --case 案例名`
3. 查看消费者报告输出，确认行业建议是否符合预期
4. 如果不符合，继续调整，重复步骤 2-3

### 场景 4：批量验证多个案例

**问题：** 需要验证规则调整对多个案例的影响。

**解决方案：**
```bash
# 批量验证多个案例
for case in 蔡蔡 张三 李四; do
  echo "验证案例: $case"
  node verify-rule.mjs --case $case > output/$case.json
done

# 对比所有案例的输出
diff output/
```

## 最佳实践

1. **小步迭代：** 每次只调整一个规则，立即验证，避免一次性修改多个规则导致问题难以定位。

2. **保存基线：** 在修改规则前，先保存当前版本的输出作为基线，方便对比。

3. **使用对比工具：** 使用 `--diff` 选项对比修改前后的输出，快速定位差异。

4. **关注覆盖率：** 定期使用 `--coverage` 选项检查规则覆盖率，确保规则被正确使用。

5. **检测冲突：** 定期使用 `--conflicts` 选项检测规则冲突，避免规则冲突导致问题。

6. **版本管理：** 将规则配置文件纳入版本管理，方便回滚和历史追溯。

7. **文档记录：** 在修改规则时，记录修改原因和预期效果，方便后续维护。

## 故障排查

### 问题 1：修改规则后没有生效

**可能原因：**
- 规则加载器缓存未清除
- JSON 格式错误

**解决方案：**
1. 检查 JSON 格式是否正确
2. 重启应用或清除缓存
3. 使用 `--coverage` 检查规则是否被加载

### 问题 2：验证工具报错

**可能原因：**
- 案例数据不存在
- 规则文件路径错误

**解决方案：**
1. 检查案例数据是否存在
2. 检查规则文件路径是否正确
3. 查看错误日志，定位问题

### 问题 3：输出结果不符合预期

**可能原因：**
- 规则配置错误
- 规则优先级错误

**解决方案：**
1. 使用 `--diff` 对比修改前后的输出
2. 使用 `--conflicts` 检测规则冲突
3. 检查规则优先级是否正确
