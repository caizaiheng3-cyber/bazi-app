## 一、项目概述

### 核心定位
帮命理师**一键生成高质量消费者报告**的双边八字排盘工具。

### 核心竞争力
排盘工具解决"算"的问题已是红海；本产品解决的是**"算完之后，怎么高效地把结果变成客户买单的报告"**——从专业推断到通俗表达的翻译。

### 产品工作流
```
消费者输入出生信息 + 选择关心领域
    ↓
系统自动排盘 + 生成双版本报告
    ↓
命理师审核专业版 → 通过后推送消费者版给用户
```

### MVP 策略
先做命理师端（专业模式），再扩展消费者端。

## 三、关键设计决策（已确认，不要重新讨论）

### 3.1 产品方向
- **方向三：命理师 + 消费者双边平台**（已确认）
- 不是纯工具、不是纯 C 端产品、不是教育产品

### 3.2 消费者报告的核心原则

**报告结构**：共情 → 解释 → 出路（三层叙事）

**命理特质五要素**（消费者报告必须包含，区别于心理分析报告）：
1. **命格意象**：用纳音给用户一个形象化"命理人设"（如"你的命属炉中火"）
2. **时间节奏**：用大运/流年给出确定性时间节点（如"2028年是黄金窗口"）
3. **术语点缀**：约 10% 的命理术语自然融入，增强专业感
4. **天命感**：归因于命格而非个人缺陷（"不是你不好，是命格使然"）
5. **具体指引**：幸运颜色/方位/数字/行业/贵人画像等命理独有建议

### 3.3 已确认的技术选型（不要改变）
- **React + TypeScript + Vite**
- **Ant Design + Tailwind CSS**
- **ECharts**（图表）
- **lunar-javascript**（历法转换）
- **Zustand**（状态管理）
- **纯前端部署**（Vercel / GitHub Pages）

### 3.4 已确认的业务规则
- 日期范围：1900-2100
- 时辰输入：双模式（具体时间 + 时辰下拉）
- 子时处理：默认早子时换日（可切换）
- 月柱规则：以节气定月（非农历初一）
- 真太阳时：高级选项，默认关闭
- 仅中文，不做国际化
- 历史记录用 localStorage，最多 20 条

## 四、技术架构速览

```
src/
├── core/                    # 核心引擎（纯逻辑，无 UI 依赖）
│   ├── calendar/            # 历法转换（公历↔农历、节气、真太阳时）
│   ├── paipan/              # 排盘计算（年/月/日/时柱、大运）
│   ├── analysis/            # 分析引擎（五行/十神/旺衰/用神/格局/神煞）
│   ├── report/              # 报告生成（专业版+消费者版+意象+开运指南）
│   └── data/                # 静态数据表（天干/地支/藏干/纳音/十神/神煞）
├── components/              # React 组件
│   ├── InputForm/           # 输入表单
│   ├── ProfessionalView/    # 专业模式视图
│   ├── ConsumerView/        # 消费者模式视图
│   └── common/              # 公共组件
├── store/                   # Zustand 状态管理
├── styles/                  # 样式
└── utils/                   # 工具函数（导出/存储）
```

**核心原则**：
- `core/` 是纯逻辑代码，不依赖 React，可独立测试
- 所有命理规则以数据表形式存储在 `core/data/`
- 报告文案模板化管理，支持复用和定制
