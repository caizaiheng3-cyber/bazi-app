## 0. 设计总纲（一图看懂）

```
┌─────────────────────────────────────────────────────────────────┐
│  📦 数据层（不变 / 引擎驱动）                                      │

                     │ 同一份引擎数据，三种视图组装
       ┌─────────────┼─────────────────────────┐
       ▼             ▼                         ▼
   🔬 v4 完整版    🧙 命理师版 v5             🎯 消费者版 v1
   （调试 / 调参） （专业用户 / 高密度）       （C 端用户 / 高情绪）
```

**核心设计原则**

- **数据一份，视图多份**：引擎层只算一次，三个产品形态消费同一份中间数据
- **事件双层模型**：所有"分析结论"都归类为「命格事件（一生级）」或「时段事件（带时间锚点）」
- **依据可追溯**：每条结论必须能反向链接到 v4 的某一段命理推理（命理师版强制展示，消费者版折叠）
- **现状贯穿**：所有趋利避害建议必须绑定命主当下现状（蔡命=33岁/IT/华东/谈婚论嫁/亚健康）

## 2. 事件双层模型（核心数据契约）

### 2.1 两种事件类型的定义

| 字段 | 🌌 命格事件（LifeEvent） | ⏰ 时段事件（TimedEvent） |
|---|---|---|
| **本质** | 命局 + 大运决定的"一生级"趋势 | 大运 × 流年 / 流月触发的"窗口期"事件 |
| **时间属性** | 跨度几十年 / 一生 | 月-年级别，必带起止时间 |
| **驱动力** | 命局结构 + 大运结构 | 流年/流月与命局的临时互动 |
| **排序方式** | 按 `netForce` 累加值排序 | 按"距今时间近 → 远"排序 |
| **示例** | "中年转运 / 婚姻偏晚 / 财靠50后" | "2026 完婚窗口 / 2031 健康警报" |
| **主舞台** | 整体视角 章节 | 流年视角 章节 |

### 2.2 统一事件结构（DataContract）

```typescript
interface BaseEvent {
  id: string;                  // 唯一 ID，便于交叉引用
  type: 'life' | 'timed';      // 事件双层类型
  title: string;               // 一句话标题（如"中年转运"/"2026 完婚窗口"）
  domain: Domain[];            // 影响领域（多选，枚举见下）
  polarity: 'good' | 'bad' | 'mixed';  // 利好 / 利空 / 混合
  intensity: 1 | 2 | 3 | 4 | 5;        // 影响强度
  evidences: Evidence[];       // 命理依据（可追溯到 v4 的章节）
  actions: Action[];           // 趋利避害建议（时段×对象×动作）
}

interface LifeEvent extends BaseEvent {
  type: 'life';
  netForce: number;            // 净影响力（驱动排序）
  primaryStage: '少年' | '青年' | '中年' | '晚年'; // 主要影响人生阶段
}

interface TimedEvent extends BaseEvent {
  type: 'timed';
  startDate: string;           // YYYY-MM
  endDate: string;             // YYYY-MM
  triggerSource: 'dayun' | 'liunian' | 'liuyue'; // 触发源
  triggerDetail: string;       // 触发机制（如"癸卯流年比劫透干夺财"）
}

interface Evidence {
  v4Section: string;           // 反向链接：v4 的章节锚点
  oneLineSummary: string;      // 一句话依据
  detailMarkdown?: string;     // 详细推理（可折叠）
}

interface Action {
  timing: string;              // 时段（如"2026.Q2 内"）
  target: string;              // 对象（如"未婚妻 / 父母 / HR / 自己身体"）
  action: string;              // 具体动作（如"领证 + 双方家庭旅行验证"）
  priority: 'P0' | 'P1' | 'P2';
}

// 领域枚举（与 v4 §4 完全对齐）
type Domain =
  | '婚姻'      // v4 §4.1
  | '财富'      // v4 §4.2
  | '事业'      // v4 §4.3
  | '健康'      // v4 §4.4
  | '六亲'      // v4 §4.5（父母/兄弟姐妹/子女）
  | '学业';     // v4 §4.6

// v4 章节锚点（Evidence.v4Section 的合法值，用于反向追溯）
type V4SectionAnchor =
  | '§1.1 旺衰' | '§1.2 用神' | '§1.3 格局' | '§1.4 十神'
  | '§2.1 合冲刑害' | '§2.2 神煞'
  | '§3.1 大运分段' | '§3.2 关键流年'
  | '§4.1 婚姻' | '§4.2 财富' | '§4.3 事业'
  | '§4.4 健康' | '§4.5 六亲' | '§4.6 学业'
  | '附录A 流年表' | '附录B 校验来源';
```

### 2.3 事件 → 视图的渲染矩阵

| 视图 | 命格事件 | 时段事件 |
|---|---|---|
| **整体视角**（命理师版） | ✅ 主舞台，按 netForce 排序 | ❌ |
| **关键事件 · 事件视图**（命理师版默认） | ✅ TOP N 一生级 | ✅ TOP N 近 10 年 |
| **关键事件 · 六亲视图**（命理师版可切） | 按 domain 分组 | 按 domain 分组 |
| **流年视图**（命理师版） | ❌ | ✅ 全部，按时间排序 |
| **六亲视图**（消费者版唯一） | 按 domain 分组 + 简化 | 按 domain 分组 + 仅近 5 年 |

## 3. 流年流月分级规则

### 3.1 流年三档分级

| 档位 | 判定规则 | 蔡命对应年份 | 命理师版呈现 | 消费者版呈现 |
|---|---|---|---|---|
| **🔴 重点年（深度展开）** | 满足任一：① 大运起止首尾年 ② 流年与命局/大运形成关键合冲刑害 ③ 流年引动主用神/主忌神 ④ 距今 ≤ 7 年 | 2027 丁未 / 2028 戊申 / 2029 己酉 / 2030 庚戌 / 2031 辛亥 / 2032 壬子 / **2033 癸丑（己未运启动）** / 2034 甲寅 / 2035 乙卯 | 单独成卡：触发机制 + 影响领域 + 季度走势 + 趋利避害 | TOP 3 仅一句话：年份 + 主题 + 一个动作 |
| **🟡 普通年（简化）** | 不命中重点年规则，但落在 27 年总表内 | 2036 丙辰 / 2037 丁巳 / ... / 2050 庚午 | 表格行：年份 + 干支 + 一句话定性 | 不展示，可点击"查看全表"折叠 |
| **⚪ 远景年（仅附录）** | 距今 > 27 年 | 2051+ | 仅附录 A 一行 | 不展示 |

### 3.2 流月触发型分级（仅当年 / 命理师版有，消费者版只取 TOP 1-2）

**触发规则**（命中任一即"重点月"）：

| 触发器 | 命理判定 | 示例 |
|---|---|---|
| **T1 · 合冲刑害触发** | 流月干支与「命局四柱 / 当前大运 / 当前流年」任一形成新的合/冲/刑/害 | 2026 年 4 月（壬辰）→ 辰戌冲日支夫妻宫 |
| **T2 · 用忌神引动** | 流月干支属于主用神（火土）或主忌神（金水）| 2026 年 5 月（癸巳）→ 癸劫财再现 + 巳火透出，冲突月 |
| **T3 · 核心宫位互动** | 流月与六领域核心宫位（年/月/日/时柱）发生干支互动 | 2026 年 11 月（己亥）→ 己土合月支亥水（化合洩月令） |

**预期密度**：每年 4-6 个重点月，其余 6-8 个月归"☁️ 平稳期"折叠。

### 3.3 过往 5 年「事件触发型」回测窗口

不按"年"切，按"命理触发节点"切——过去 5 年（2021-2025）蔡命命理上的关键触发节点：

| 节点 | 时间锚点 | 触发机制 | 应验领域 |
|---|---|---|---|
| **大运启动期** | 2023.Q1（虚岁 31 入庚申）| 第 3 步大运由壬戌切换至庚申（金生水加重，仍属忌神）| 事业方向调整 / 心理摇摆 |
| **食神透干年** | 2024 全年（甲辰）| 月支亥中食神甲透出 + 辰冲戌（短暂打开戌库 → 戊土短暂复活）| 表达/创作/副业起势 |
| **伤官见火年** | 2025.Q2-Q4（乙巳）| 时支卯本气乙透出 + 巳火激活戌中丁 | 表现欲爆发 / 与权威冲突 |
| **比劫透火年** | 2026 当下（丙午）| 丙火透 + 午火合戌（午戌半合火局），但日干壬自坐戌 | 火气浮现 / 婚姻窗口 |

→ §4 章节会把这 4 个节点写成「事后叙述卡片」。

## 4. 过往 5 年「关键节点卡片」结构（避巴纳姆）

### 4.1 卡片必备字段（所有节点都要齐全 6 项，缺一不可）

| 字段 | 说明 | 反巴纳姆要点 |
|---|---|---|
| **① 季度锚点** | YYYY.QN（精确到季度） | 必须具体，禁用"那段时间" |
| **② 命理触发** | 干支机制一句话（如"庚申大运启动 + 流年癸卯比劫透干"）| 必须给出**机制名**，不是泛指 |
| **③ 应验领域** | 必须从 6 个 Domain 中选 1-2 个 | 禁止"人生/命运"这种空泛词 |
| **④ 推断现象** | "你应该经历了 X"——X 必须是**具体可对照的行为/情绪/事件类型** | 给 3 个候选现象，让命主自勾对应几条 |
| **⑤ 强度评分** | 1-5 颗星（基于 netForce 推算） | 量化避免"大概可能" |
| **⑥ 与未来的关联** | 这个节点为接下来 5 年埋下了什么伏笔 | 把回测和前瞻打通 |

### 4.2 反巴纳姆判定标准（卡片质量检查）

每张卡片产出后必须通过以下 3 个反问：

- **R1 · 替换性测试**：把这张卡片套到另一个完全不同的人（如女性、80 后、不同行业）身上，是否仍然"看起来挺准"？→ 如果是，**违规，退回重写**
- **R2 · 现象具体度**：卡片里"你经历了 X"中的 X，是否能用一个具体动词 + 具体场景描述？（如"换了工作"vs "事业有变化"）→ 必须是前者
- **R3 · 时间精度**：是否精确到季度？→ 笼统的"那一年"不达标

### 4.3 卡片样例骨架（蔡命真实数据）

```
🃏 节点卡 · 2023.Q1「庚申大运启动」
─────────────────────────────
② 触发：第 3 步大运由壬戌切换至庚申（金生水加重，仍属忌神，
       但申中藏戊土微补）+ 流年癸卯比劫透干（卯戌合更稳，
       戊土被夺更彻底）
③ 领域：[事业] [六亲]
④ 推断现象（3 条对照）：
  □ 你应该在 2023 年初经历了一次重大方向选择
    （跳槽 / 转岗 / 项目方向决策 / 创业犹豫 任选其一）
  □ 决策过程中出现"明明想要 A 却被推向 B"的拉扯感
  □ 决策后的 6 个月你内心持续质疑这个决定（庚申偏印致疑）
⑤ 强度：⭐⭐⭐⭐ （v4 §3.1 大运分段权重 75 ·
       庚申忌神运启动 + 流年比劫卯戌合双重触发）
⑥ 伏笔：这次方向选择，决定了你 2024-2026 这 3 年的能量分配赛道
       —— 也直接决定 2033 己未运启动时你"接不接得住主升浪"
```

### 5.2 双视图切换的交互契约

**事件视图（默认）**：
- 卡片流，每张卡片头部带「🌌 命格 / ⏰ 时段」徽章
- 排序：`命格事件按 netForce 倒序` 在上、`时段事件按时间近→远` 在下
- 每张卡：`title` + `domain 标签组` + `polarity 色彩` + `intensity 星级` + `evidences 折叠区` + `actions 三维表`

**六亲视图（切换）**：
- 6 个领域 Tab：`婚姻 | 财富 | 事业 | 健康 | 六亲 | 学业`
- 每个 Tab 下：该领域的命格事件 + 时段事件混排
- 同一事件影响多个领域时**在每个相关领域 Tab 都出现一次**（带"亦影响 XXX"标注）

### 5.4 命理师版必须保留 v4 的 5 个"硬核"特征

- ✅ **位置力学 5 维**：柱位 + 宫位 + 距离 + 触发模式 + 净影响力（在事件依据折叠区原样保留）
- ✅ **置信度评级**：每个核心结构带 ⭐ 评级（沿用 v4 §1.x 的"权重/置信"标签）

### 6.2 消费者版的 5 条"减法 + 加法"规则

**减法（拿掉）**：
- ❌ 删除所有命理术语（"戌卯六合"、"伤官"、"netForce" 等命理黑话一律不出现在主视图）
- ❌ 删除所有数字量化（不出现 `−8.4`、`+1.6` 这类）
- ❌ 删除依据折叠区（仅显示"基于 N 条命理依据"标签，不可展开）
- ❌ 删除流年全表（只展示当年 + TOP 3 关键流年）
- ❌ 删除事件视图（强制六亲视图）

**加法（增加）**：
- ✅ 故事化叙事（"你的人生像一艘…"这种比喻语言）
- ✅ 强情绪表达（"卧槽"、"心安"、"千万别"等口语化标签可用）
- ✅ 单动作聚焦（每个领域只给 1 个最高优先级动作，不给三维表）
- ✅ 视觉化牌面（5 张牌沿用 v4 §5 的塔罗牌形态）
- ✅ 升级钩子（每个领域底部一句"想知道更详细的依据？→ 升级命理师版"）

### 6.3 消费者版的"故事化改写"规范

每个领域必须改写为**「现状 → 走势 → 关键时点 → 一个动作」**四段式：

```
💍 婚姻 · 你的故事

[现状]
你这辈子的婚姻像一锅慢炖的暖汤——前 30 年水太凉，
但你偏偏是命中带"火苗"的人，所以这碗汤一定要靠"对的人"才能煮开。

[走势]
2026 年是你命书里写明的"火苗最旺的一年"，是这辈子最适合完婚的窗口。
错过这一年，下一个同等强度的窗口要等到 2034。

[关键时点]
2026.Q2 之前完成领证；2027 安顿；2030 前后生育第一胎。

[今年要做的 1 件事]
🎯 在 2026 年 6 月底前完成婚姻登记。
   （不办仪式可以，但法律关系要先锁定。）
```

→ 同一份事件数据，在命理师版是 `actions[]` 三维表，在消费者版是这样一段散文。
