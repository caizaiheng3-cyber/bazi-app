| 外显 + 实质② | 命主自己也无感 | "你身体的命理本能在替你筛选"（命主自己不知道有这套筛选机制） |
| 触发 · 流年引动 | 2026 丙午 | "2026 年正好是" |
| 触发 · 等运激活 | 41 岁己未（兜底） | "下一个等到 2037 还要等 11 年" |
| 依据 | — | 底部"📚 5 条" 不可展开 |

#### 2.4.5 写作技法记录（事件轴通用结构）

```
共情（≤30 字）
  把用户的"困扰"翻译成"命里的设计"
  例："你不是挑剔，是命里有一个非常具体的'对的人'标准"

命理判断（2-3 句）
  命理上这件事的"性质" + 时间窗口

🎯 趋利避害（1-3 条）
  每条 = 时间锚点 + 对象指代 + 动作动词

📚 基于 N 条命理依据
```

## 3. 模拟过程暴露的 3 个问题

> 走完一遍模拟后，发现《信息选择设计-v1.md》本身需要补充 3 件事。建议合并入下一版（v1.1）。

### 3.1 问题 1 · 命理师视角的"建议草稿"严重依赖 GAP-1

**现象**：第 1 屏总论草稿里，"33 岁的蔡先生" "互联网" "未婚" 这些贴身指代——目前 `basicInfo` 里只有姓名/性别/生辰，**职业/婚姻状态/所在地/年龄阶段都没有**。

**影响**：

- 命理师视角的"草稿区"（§2.5）退化为**模板填空**——所有需要"贴身指代"的位置都得手填
- 消费者视角的"总论"（§3.1）失去戳点——"33 岁正卡在想结婚但又怕结错"这种句子根本生不出来

**对应缺口**：GAP-1（详见 `信息选择设计-v1.md` §5.1）

**建议**：在 InputForm 至少新增 4 个**可选**字段：
- 当前职业大类（IT / 金融 / 体制 / 教育 / 医疗 / 其他）
- 当前所在地（省市级别即可）
- 婚姻状态（未婚 / 谈婚论嫁 / 已婚 / 离异）
- 当前最关心的领域（多选 6 域）

→ 不要求强制填，但填了的话所有论断的"贴身度"会 +1 个数量级。

---

### 3.2 问题 2 · 消费者视角的"命理元素隐式翻译"需要一份对照库

**现象**：模拟时把"戌卯六合化火"翻译成"婚姻像慢炖的暖汤"——这个翻译目前是**手写产物**。

**风险**：

- 不能靠 LLM 现编（不可控、可能违反禁区）
- 不能靠手写（不可扩展、不能跑在引擎里）

**对应缺口**：原 `信息选择设计-v1.md` §5 没有列出，建议新增 **GAP-13 · 命理元素 → 消费者比喻对照库**。

**建议结构**：

```
比喻库.json
{
  "戌卯六合化火": {
    "domain": "姻缘",
    "metaphor_when_unmarried": "你这辈子的婚姻像一锅慢炖的暖汤——前 30 年水太凉，但你命里偏偏带一束火苗",
    "metaphor_when_married": "你和配偶的关系是命中注定的'火源绑定'——你身上的火苗是被对方点燃的",
    "key_words": ["火苗", "暖汤", "煮开"],
    "valid_for_ages": [18, 90]
  },
  "比劫夺财": {
    "domain": "财富",
    "metaphor": "你这辈子的钱，像一块肉被四只狼围着",
    ...
  }
}
```

**预估工作量**：核心命理元素约 30-50 条，每条 5-10 分钟手写，总计 1-2 个工作日。

---

### 3.3 问题 3 · 事件轴问答需要"标准事件库"

**现象**：蔡问"想找女朋友怎么办"——回答里的"虎/马/狗 / 4-7 月 / 夜班"这些**不是临场算的，是从用神火的属性反推的固定话术**。

**对应缺口**：GAP-5 的细化（详见 `信息选择设计-v1.md` §5.1）。

**建议**：明确"事件意图识别 → 命中标准事件库"的两段式：

```
标准事件库（按 6 域 × 3-5 个高频事件）:

💍 姻缘：
  - 找对象（择偶标准）
  - 婚姻窗口（什么时候领证）
  - 关系评估（合不合）
  - 分手 / 离婚（要不要分）

💰 财富：
  - 跳槽涨薪
  - 投资判断
  - 借贷担保
  - 大额消费（买房/买车）

🏆 事业：
  - 跳槽
  - 晋升
  - 创业
  - 老板关系

... (健康 / 家庭 / 自我 同结构)
```

每个标准事件对应一个**字段调取规则 + 回话模板**：

```
事件: 姻缘.找对象
字段调取:
  - 用神五行（→ 推配偶五行属性）
  - 配偶星位置（→ 推年龄差/性格）
  - 当前流年（→ 推窗口期）
回话模板:
  - 共情: "你不是挑剔，是命里有具体标准"
  - 命理判断: "{用神元素}属性的人是你的{比喻}"
  - 趋利避害: 优先认识 {推荐画像} / 严格回避 {风险画像} / 窗口期 {时间锚点}
```

**预估工作量**：6 域 × 4 事件 = 24 个标准事件，每个 30 分钟设计模板，总计约 2-3 个工作日。
