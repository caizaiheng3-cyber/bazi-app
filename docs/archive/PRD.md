## 一、产品定义

### 1.1 一句话描述
帮命理师**一键生成高质量消费者报告**的双边八字排盘工具。

### 1.2 核心竞争力
市面上排盘工具解决了"算"的问题，AI 批命解决了"直接给答案"的问题，但**没有工具帮命理师把专业推断高效转化为客户愿意买单的报告**——这就是本产品的唯一核心。

### 1.3 目标用户画像

| 角色 | 画像 | 核心痛点 | 产品价值 |
|------|------|---------|---------|
| **命理师**（服务端） | 兼职/副业命理师，日均接单 2-5 单，白天上班晚上接单 | 每单报告撰写耗时 30-60 分钟，且质量不稳定 | 将报告时间压缩到 5 分钟，质量稳定可复用 |
| **消费者**（用户端） | 25-40 岁，对八字感兴趣，有具体困惑（事业/感情/财运） | 免费工具看不懂，花钱找人不知道靠不靠谱 | 收到一份有命理特质、有温度、有指引的个性化报告 |

### 1.4 产品工作流

```
首次使用：低频深度报告
─────────────────────
消费者输入出生信息 + 选择关心领域
          ↓
系统自动排盘 + 生成双版本报告
          ↓
    ┌─────────────────┐
    │  专业版（给命理师） │ ←── 命理师审核/微调
    └─────────────────┘
          ↓ 审核通过
    ┌─────────────────┐
    │  消费者版（给用户）  │ ←── 推送给消费者
    └─────────────────┘

日常使用：高频命理对话（核心粘性设计）
────────────────────────────────
用户日常打开 App
          ↓
    ┌─────────────────┐
    │  每日命理 Dashboard │  今日宜忌 + 提问入口 + 历史问答
    └─────────────────┘
          ↓ 用户提问（决策/择吉/宜忌/开放）
    ┌─────────────────┐
    │   提问页（Ask）    │  场景按钮 + 自由输入 + 关联领域
    └─────────────────┘
          ↓ 系统基于命盘 + 流年流月流日
    ┌─────────────────┐
    │  命理师人设回答   │  "先生" 风格 · 共情→解释→建议三段
    └─────────────────┘
          ↓
    ┌─────────────────┐
    │   命理日记（History）│  问答时间线，沉淀为用户资产
    └─────────────────┘
```

> **核心设计**：把八字从「一次性算命」转化为「日常决策伴侣」，
> 从低频工具升级为高频陪伴产品，是用户留存和长期价值的核心。

## 三、技术方案

### 3.1 技术栈选型

| 层面 | 选型 | 理由 |
|------|------|------|
| **框架** | React + TypeScript | 组件化开发，类型安全，生态丰富 |
| **构建工具** | Vite | 极快的开发服务器，开箱即用 |
| **UI 组件库** | Ant Design / Tailwind CSS | Ant Design 提供表单/表格组件，Tailwind 做定制化样式 |
| **图表** | ECharts 或 Chart.js | 五行分布图、大运时间轴等可视化 |
| **日历/历法** | lunar-javascript（npm 包） | 支持公历转农历、节气精确计算、已覆盖 1900-2100 |
| **图片导出** | html2canvas | 将 DOM 渲染为图片 |
| **状态管理** | Zustand | 轻量，适合中小型项目 |
| **部署** | Vercel / GitHub Pages | 纯前端，无需服务器 |

### 3.3 核心设计原则

1. **引擎与 UI 分离**：`core/` 下的所有模块是纯逻辑代码，不依赖 React，方便独立测试和未来迁移
2. **规则数据化**：所有命理规则以数据表形式存储在 `data/` 目录，修改规则不需要改逻辑代码
3. **报告模板化**：消费者报告的文案模板独立管理，支持模板复用和定制
4. **推断链可追溯**：每一步分析结果都保存推断依据，供专业模式展示

|--------|------|---------|

| 十神推算 | 10 个案例 | 100% 正确 |

| 旺衰判断 | 5 个案例 | 与主流命理师判断一致率 ≥ 80% |

| 验收项 | 方法 | 通过标准 |
|--------|------|---------|
| 命理特质五要素 | 检查报告是否包含全部五要素 | 5/5 全覆盖 |
| 共情→解释→出路结构 | 检查报告结构 | 三层结构完整 |
| 术语占比 | 人工审核 | 约 10%（±5%） |
| 无恐吓措辞 | 全文检索"大凶""灾难"等词 | 0 处 |
| 文案可读性 | 给 3 个非专业人士阅读 | 均表示"能理解" |

|--------|---------|
| 排盘计算时间 | < 1 秒 |
| 报告生成时间 | < 2 秒 |
