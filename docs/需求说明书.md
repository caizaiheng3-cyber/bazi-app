## 一、项目概述

### 1.1 项目名称
八字排盘系统

### 1.2 项目目标
构建一个帮助命理师将"可靠的专业推断"高效转化为"能治愈用户情绪、具备命理专业特质的个性化报告"的双边工具。

**核心竞争力**：命理师一键生成高质量消费者报告——系统自动完成从专业推断到通俗表达的翻译工作，命理师只需审核和微调，即可交付一份专业又易懂的个性化命理报告。

**核心解决的问题**：
- 命理师端：排盘和推断过程系统化，减少手工计算和报告撰写时间（1 小时 → 5 分钟）
- 消费者端：收到的不是冰冷的术语堆砌，而是**有命理特质、有情感温度、有实用指引**的个性化报告

### 1.3 产品定位
**方向**：命理师 + 消费者双边平台

**工作流**：
1. 消费者提交出生信息 + 选择关心的领域
2. 系统自动排盘 + 生成双版本报告（专业版 + 消费者版）
3. 命理师在专业模式下审核推断准确性
4. 审核通过后，消费者报告推送给用户

**MVP 策略**：先做好命理师端（专业模式），再扩展消费者端

### 1.4 目标用户
- **服务提供者**（命理师/运营者）：需要完整的推断数据和过程，用于审核和判断产品输出的准确性
- **消费者**（终端用户）：希望看到简单易懂的分析结果，重点关注自己关心的问题领域

### 1.5 双视角输出设计理念
系统对同一份八字数据提供两种不同深度的输出视角：

**服务提供者视角（专业模式）**：
- 完整的命理推断信息和原始数据
- 关键信息的重要性标注和梳理
- 每一步推断的过程、逻辑链和依据
- 推断规则标注出处（如《子平真诠》《滴天髓》等）
- 不确定推断标记置信度等级（高/中/低）
- 用于判断产品输出是否准确、是否有遗漏

**消费者视角（用户模式）**：
- 先了解用户最关心的问题类型（事业、感情、健康、财运等），对该类问题给出更详细的输出
- 报告结构遵循"**共情 → 解释 → 出路**"三层结构
- 核心输出简单易理解，使用通俗语言
- 附带约 10% 的专业性依据辅助说明，让结果看起来更专业可信
- 避免术语堆砌，重结论轻过程

### 1.6 消费者报告的命理特质（核心差异化）

消费者报告必须区别于普通的心理分析报告，需要具备以下**命理专属特质**：

**为什么需要命理特质？**
如果报告读起来像心理咨询报告，用户没有理由来"算八字"而不是去找心理咨询师。命理报告的独特价值在于：以"天命"的视角帮用户理解自己和人生，这是心理分析无法替代的情感体验。

**与心理分析报告的核心差异**：

| 维度 | 心理分析报告 | 命理报告 |
|------|------------|---------|
| 归因方式 | 归因于原生家庭/性格/认知 | **归因于命格**——"不是你不好，是命格使然" |
| 时间维度 | 回溯过去（分析成因） | **指向未来**（什么时候运势变化、何时行动） |
| 语言体系 | 心理学术语 | **命理术语**（财运、贵人、食伤泄秀） |
| 确定性 | "你可以尝试…也许有帮助" | **"你命中注定…""这步大运会…"** |
| 意象感 | 抽象概念分析 | **具象化意象**（"命属炉中火""你像一棵扎根深处的大树"） |
| 神秘感 | 去神秘化 | **适度保留神秘感**——这是用户的情感需求 |

**消费者报告必须体现的五个命理特质要素**：

1. **命格意象**：用纳音/五行给用户一个形象化的"命理人设"
   - 示例："你的命属炉中火，是那种内心有一团火焰、越压越旺的人"
   - 作用：给用户一个独特的自我认知标签，这是命理独有的

2. **时间节奏**：用大运/流年给出确定性的时间节点
   - 示例："2028 年之后进入丁亥大运，你会明显感受到事业上的变化"
   - 作用：心理分析无法给出"几年后会怎样"，这是命理的独特价值

3. **术语点缀**：有节制地使用命理术语（约占全文 10%）
   - 示例："你八字中财星旺但印星也强"而不是"你既想赚钱又在乎精神追求"
   - 作用：让用户觉得"这是真的在看八字"，增强专业感和信任度

4. **天命感**：适度传递"这是命格特质，不是你的缺点"的暗示
   - 示例："你做事讲究完美，有时会因此给自己太大压力——这是食神格的人天生的特质，不是缺点，而是你成事的底气"
   - 作用：**帮用户卸下自我归罪的包袱**——这是命理服务最核心的情绪价值

5. **具体指引**：提供命理特有的具象化建议
   - 示例：幸运颜色、有利方位、适合行业、贵人特征等
   - 作用：给用户可执行的"仪式感"行动，心理咨询不会说"今年多穿黄色"

### 1.7 高频化设计：每日命理对话（产品粘性核心）

**为什么需要高频化？**
传统八字工具是「一次性消费」——用户算完一次，3 个月不会再来。这导致 LTV 极低、付费意愿弱。
本产品通过「每日命理对话」把八字从**低频深度报告**升级为**高频日常陪伴**，让用户每天打开 App。

**核心价值主张**：
> 整体报告告诉你"你是什么样的人"；每日对话告诉你"今天该怎么办"。

**功能形态**：

1. **每日 Dashboard（习惯入口）**
   - 顶部今日宜忌卡（基于今日干支 + 原局用神/忌神计算）
   - 中部"向命盘提问"输入框 + 4 类场景快捷按钮
   - 下部本周运势曲线 + 历史问答时间线

2. **4 类提问场景**：
   | 场景 | 示例问题 | 命理依据 |
   |------|---------|---------|
   | **决策建议** | "今天该不该去面试？" | 今日干支 vs 原局喜忌 + 大运 + 月令 |
   | **时机择吉** | "什么时间签合同最好？" | 未来 7 天逐日干支评分 |
   | **每日宜忌** | "今天宜做什么？" | 今日五行 + 用神配合度 |
   | **开放问答** | 任意问题 | 关键词识别 → 映射到命理维度 |

3. **回答形态：命理师人设对话（"先生"风格）**
   - 自称"先生"，语气温和但有权威感
   - 三段式结构：**共情**（理解你的处境）→ **解释**（命理上为什么）→ **建议**（具体怎么做）
   - 视觉：宋体大字 + 古典对话气泡 + 关键判断印章式标签（宜/忌/慎）
   - 必含命理依据：流年/流月/流日 + 原局用神 + 当前大运
   - 保留约 10% 术语原则（不能堆砌也不能没有）

4. **命理日记（用户资产）**
   - 时间线展示所有历史问答
   - 可标记重要、回看、对比"先生当时怎么说"和"实际怎么样"
   - 形成用户的**个人命理档案**，使用越久价值越高（增加迁移成本）

**设计禁区**：
- ❌ 禁止用"今日运势 100 分"这种伪科学评分
- ❌ 禁止恐吓措辞（"大凶""灾祸"等）
- ❌ 禁止脱离命盘的泛泛建议（每条建议都必须能追溯到命理依据）
- ❌ 禁止鸡汤化（必须有命理特质，不是心灵鸡汤）

## 二、功能需求

### 2.1 八字计算（核心）
- [ ] 用户输入：出生年、月、日、时、性别
- [ ] 输入方式：公历日期输入，系统自动转换为农历
- [ ] 时辰输入：支持两种方式 —— ① 具体时间输入（如 14:30），系统自动判断时辰；② 时辰下拉选择（子时、丑时...），方便只知道大概时辰的用户
- [ ] 输出四柱：年柱、月柱、日柱、时柱（天干 + 地支）
- [ ] 支持日期范围：**1900 年 ~ 2100 年**
- [ ] 性别输入：用于大运排盘方向判断
- [ ] 子时处理：默认采用"早子时换日"方案（23:00-0:00 为夜子时属当日，0:00-1:00 为早子时属次日），高级设置中可切换为"子时不换日"流派
- [ ] 月柱规则：以**节气**定月柱（非农历初一），闰月不影响八字计算，但展示农历日期时需正确标注闰月
- [ ] 真太阳时：作为高级选项，默认关闭。开启后可选择出生省市（内置经度数据），系统自动修正为真太阳时

### 2.2 五行分析
- [ ] 统计八字中金、木、水、火、土的数量分布
- [ ] 天干五行归属
- [ ] 地支五行归属（含地支藏干的五行）
- [ ] 五行强弱判断
- [ ] 五行缺失提示
- [ ] 可视化展示五行分布（如图表）

### 2.3 十神分析
- [ ] 以日干为中心，推算其余七个天干地支的十神关系
- [ ] 十神类型：正官、偏官（七杀）、正印、偏印（枭神）、比肩、劫财、食神、伤官、正财、偏财
- [ ] 十神含义简要说明
- [ ] 十神格局初步判断

### 2.4 大运排盘
- [ ] 根据性别和年柱判断大运顺逆
- [ ] 计算起运年龄
- [ ] 排列每步大运（每步 10 年）的天干地支
- [ ] 显示大运起止年份
- [ ] 流年展示

### 2.5 纳音分析
- [ ] 六十甲子纳音对照
- [ ] 显示四柱各柱的纳音五行
- [ ] 纳音含义简要说明

### 2.6 神煞查询
- [ ] 常见神煞查询（天乙贵人、文昌贵人、驿马、桃花、华盖等）
- [ ] 神煞查询规则：基于年柱/日柱查四柱
- [ ] 神煞含义简要说明
- [ ] 吉神与凶煞分类展示

### 2.7 双视角输出

#### 2.7.1 服务提供者视角（专业模式）
- [ ] 完整四柱排盘数据（含藏干、纳音、十神标注）
- [ ] 五行力量量化评分（天干得分 + 地支得分 + 藏干得分 + 月令旺衰）
- [ ] 日主强弱判断：展示判断依据链（得令/失令 → 得地/失地 → 得生/受克 → 综合结论）
- [ ] 用神/忌神推断过程：展示从日主强弱到用神选取的完整逻辑
- [ ] 十神格局分析：格局判定 + 成格/破格条件 + 判断依据
- [ ] 大运流年逐步分析：每步大运与原局的作用关系、吉凶判断及理由
- [ ] 神煞完整列表：含神煞来源（由哪柱查哪柱得出）
- [ ] 关键信息标注：用颜色/图标标记影响命局的关键因素（如格局关键字、用神位置）
- [ ] 信息可信度标注：对不确定的推断标记置信度等级（高/中/低）
- [ ] 原始数据导出：支持导出完整的计算中间数据

#### 2.7.2 消费者视角（用户模式）

**交互输入**：
- [ ] 关心领域选择：用户可选择最关心的问题类型（事业、感情、健康、财运、学业、人际）

**报告结构（共情 → 解释 → 出路）**：
- [ ] 命格意象开篇：用纳音/五行给用户一个形象化的"命理人设"（1-2 句话），作为整份报告的"人设锚点"
- [ ] 性格特质概述：基于十神格局，用通俗语言 + 天命感表达描述命主性格（"这是你命格天生的特质"）
- [ ] 重点领域详细分析：针对用户选择的关心领域，提供 3-5 个要点，融入时间节奏（大运/流年节点）
- [ ] 其他领域简要概述：非重点领域各用 1-2 句话概括
- [ ] 流年运势提示：当前年份和未来 1-2 年的运势趋势，给出**确定性的时间预期**
- [ ] 命理术语点缀（约 10%）：在关键结论后附带简短的命理依据，使用真实术语而非心理学表述
- [ ] 幸运元素推荐：幸运颜色、方位、数字、适合行业、贵人特征等命理特有的具象化建议
- [ ] 注意事项提醒：用"天命感"语言表达风险提示（"这段时间命格提醒你…"），不使用"大凶"等恐吓性措辞
- [ ] 温暖收尾：积极正向的结语，传递"命运在自己手中"的力量感

### 2.8 可视化界面
- [ ] 美观现代的 UI 设计
- [ ] 输入表单：日期时间选择 + 时辰选择 + 性别选择
- [ ] 高级设置面板：真太阳时开关、出生地选择、子时流派切换
- [ ] 排盘结果展示：四柱清晰排列
- [ ] 五行分布图表
- [ ] 十神关系展示
- [ ] 大运时间轴
- [ ] 神煞标签展示
- [ ] 响应式布局（兼容桌面和移动端）
- [ ] 导出功能：支持将排盘结果导出为图片（方便社交分享）

### 2.9 历史记录
- [ ] 使用浏览器 localStorage 保存最近 20 条查询记录
- [ ] 记录内容：输入信息 + 排盘结果摘要
- [ ] 支持查看历史记录、重新加载、删除
- [ ] 页面加载时清理异常中间状态（防御性处理）

### 2.10 每日命理对话（高频化模块 · 对应 1.7 设计层）

> 本节为字段级需求，概念与价值参见 1.7。详细输出形态参见《输出模板.md》模板三。

#### 2.10.1 流年流月流日推算
- [ ] **流日干支**：输入任意公历日期（1900-2100），输出当日年/月/日干支三柱（流时按需）
- [ ] **节气定月**：跨节气日的流月切换正确（与排盘引擎月柱算法共用）
- [ ] **批量推算**：支持输入起始日 + N 天，输出 N 日干支序列
- [ ] **配合度评分**：每日相对原局输出 `{宜值: 0-100, 忌值: 0-100, 慎值: 0-100}` 三元组，评分规则可追溯到具体命理依据

#### 2.10.2 提问意图解析
- [ ] **场景类型**：4 选 1 — 决策建议 / 时机择吉 / 每日宜忌 / 开放问答
- [ ] **领域识别**：从自由文本中识别 6 类领域 — 事业 / 感情 / 财运 / 健康 / 学业 / 人际（开放问答场景准确率 ≥ 80%）
- [ ] **时间归一化**：识别"今天/明天/本周末/下周一/5 月初"等表述并转为具体日期
- [ ] **提问输入字段**：
  | 字段 | 类型 | 必填 | 说明 |
  |------|------|------|------|
  | 场景类型 | enum | 是 | 4 类场景之一 |
  | 提问文本 | string(≤500) | 是 | 用户自由输入 |
  | 涉及日期 | date | 否 | 默认今天，择吉场景为时间窗口 |
  | 领域标签 | enum | 否 | 影响命理维度选取 |

#### 2.10.3 每日宜忌生成
- [ ] **宜/忌/慎三标签**：每日各输出 2-4 条具体事项（如"立约""远行""借贷"），不输出"今日运势 X 分"
- [ ] **吉时凶时**：12 时辰逐一评分，输出当日最佳/最差时辰
- [ ] **先生一句话**：基于当日最高优先级判断，1 句白话总览（≤30 字）
- [ ] **规则库**：内置 30+ 条宜忌规则（如"用神得令日宜立约""忌神冲日忌远行"），可扩展

#### 2.10.4 "先生"对话回话引擎（核心）
- [ ] **三段式结构**：每条回答必须包含 共情 / 解释 / 建议 三段，缺一不可
- [ ] **命理依据必填**：每条回答自动拼接 流日干支 + 原局喜忌 + 当前大运 三项依据，缺项时引擎报错而非降级输出
- [ ] **印章标签**：宜（朱砂红 `#C5392F`）/ 忌（水墨黑 `#2C2A28`）/ 慎（金色 `#B8860B`）三档
- [ ] **术语 10% 原则**：自动统计回话中术语占比，落在 5%-15% 区间为合格，超出范围标记待人工校审
- [ ] **设计禁区拦截**（硬性）：
  - 禁止"大凶 / 灾祸 / 血光 / 100 分 / 满分 / 运势评分"等违禁词，命中即拦截
  - 禁止脱离命盘的泛泛建议（每条建议必须能追溯到命理依据）
  - 禁止鸡汤化（必须含命理特质）
- [ ] **健康/法律/重大投资免责**：命中相关关键词时自动追加免责提示（"命理是参照，不是替代"）

#### 2.10.5 命理日记（用户资产）
- [ ] **持久化**：每条问答写入 localStorage，结构化存储
- [ ] **数据结构**（每条记录）：
  | 字段 | 类型 | 说明 |
  |------|------|------|
  | id | uuid | 唯一标识 |
  | createdAt | timestamp | 提问时间 |
  | scene | enum | 场景类型 |
  | domain | enum | 领域 |
  | question | string | 原始提问 |
  | reply | object | `{empathy, explanation, advice, label}` 四字段 |
  | basis | object | `{dailyGanzhi, originLikes, currentDayun}` 命理依据快照 |
  | starred | bool | 是否收藏 |
  | feedback | enum/null | 实际反馈：应验 / 未验 / 部分应验 / 待回填 |
- [ ] **时间线**：倒序展示历史问答，支持按 场景 / 标签 / 收藏 筛选
- [ ] **追问**：可对任一历史问答发起追问，自动继承上下文
- [ ] **导出**：支持导出本月所有"先生回话"为图片或 Markdown
- [ ] **容量**：localStorage 最多保存 200 条对话记录，超出按时间倒序淘汰（收藏的不淘汰）

## 三、非功能需求

### 3.1 性能要求
- 排盘计算响应时间 < 1 秒
- 页面首屏加载时间 < 3 秒

### 3.2 兼容性要求
- 支持主流现代浏览器（Chrome、Firefox、Safari、Edge）
- 响应式布局，兼容桌面端和移动端
- 语言：**仅中文**（八字为中国传统文化，术语翻译难度大，暂不做国际化）

### 3.3 数据准确性
- 八字计算需要准确可靠，涉及历法转换需经过验证
- 节气换月规则需正确实现（八字以节气分月，非农历初一）
- 真太阳时修正需基于可靠的经度数据

### 3.4 数据存储
- 纯前端方案，无需后端服务器
- 用户数据（历史记录）存储在浏览器 localStorage 中
- 需考虑 localStorage 容量限制，避免存储过大数据
